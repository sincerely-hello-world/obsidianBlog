
最佳实践是为每个新的工作空间创建一个新的目录。

目录名称无关紧要，但是有助于表明工作空间的目的。

> 一个最佳实践是将工作空间中的任何包放置在`src`目录中。
+ 目录名为`workspace_folder`，表示`开发工作空间`：
+ 在`workspace_folder/src` 创建自己的 `软件包`

## 工作区
构建前
```
./chapt2_ws/
	└── src
```
构建后：
```
./chapt2_ws/
	├── build
	├── install
	├── log
	└── src
```

 运行工作区 下的节点
```bash
source /opt/ros/foxy/setup.bash
cd ./chapt2_ws/ 
source install/local_setup.bash
ros2 run 
```

## 工作区结构

单个工作区：
```bash
workspace_folder1/
    src/
      cpp_package_1/
          CMakeLists.txt
          include/cpp_package_1/
          package.xml
          src/          # 源码文件

      py_package_1/
          package.xml
          resource/py_package_1
          setup.cfg
          setup.py
          py_package_1/  # 源码文件
      ...
```
多工作区
```
chapt3/
├── topic_practice_ws #   工作空间ws1
│   └── src
│       ├── status_display # ws1的 包1
│       ├── status_interfaces
│       └── status_publisher
└── topic_ws          #   工作空间ws2
    └── src
        ├── demo_cpp_topic # ws2的 包1 ...
        └── demo_python_topic
```
